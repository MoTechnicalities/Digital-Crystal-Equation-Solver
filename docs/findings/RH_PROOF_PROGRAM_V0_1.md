# RH Proof Program (v0.1)

Status: Active research pipeline, not prize-ready.

## Purpose

Define a strict progression from exploratory geometric-logic evidence to a prize-eligible RH proof attempt.

## Prize Requirement Anchor

To satisfy the Millennium Prize bar, one must provide either:
- a correct proof that every non-trivial zeta zero has real part `1/2`, or
- a correct disproof via a rigorously validated non-trivial zero off the critical line,

with full community verification.

## Pipeline Stages

1. Formal semantics layer
- maintain and refine PSI/EI definitions
- keep theorem candidates explicit (T1/T2/T3 and RH-specific extensions)

2. Deterministic evidence layer
- witness catalogs from reproducible runs
- transition and asymmetry maps with confidence intervals

3. Proof-obligation layer
- explicit obligation registry
- objective satisfied/open states tied to artifacts

4. RH-specific theorem chain
- convert RH equivalent statements into dependency graph
- prove intermediate lemmas with full assumptions and failure criteria

5. Manuscript and verification
- assemble complete proof candidate
- independent checks, adversarial review, and formal correction cycles

## Current Program Position

- Strong on stages 1 and 2.
- Initial automation for stage 3 is now active.
- Explicit API signature hard gate is implemented:
	- `path_signature` and `endpoint_signature` are first-class `/v1/csif/math` fields.
	- T1/T2/T3 conformance checks are wired against those fields in Rust tests.
	- RH pipeline obligation `O5-explicit-signatures` is now machine-checkable and currently satisfied.
- Stages 4 and 5 remain open.

## Operating Principle

Geometric logic remains core:
- constraints define admissible paths
- paths define signatures
- signatures become machine-auditable evidence

But evidence is not proof. The pipeline is designed to bridge that gap without overstating results.
